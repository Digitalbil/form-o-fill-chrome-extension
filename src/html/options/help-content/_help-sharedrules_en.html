Sharing field definitions

It is possible to share field definitions between rules.
This enables you to write a set of field definitions once and insert those somewhere in another rule's workflow.

var rules = [{
This is the name the shared rule will be referenced by
  name: "these field definitions are shared",
Since every shared rule can be used independently,
it's good to use a "url" or "content" matcher
  url: "http://www.like-every-other-rule.com",
  fields: [{
Normal field definitions follow.
    selector: "input[type=text]",
All field definitions are imported when the rule is referenced.
    value: "like always"
  }]
};

var rules = [{
This is a rule that uses the shared rule from above.
It can be defined in whatever tab you want
  name: "using the shared rule from above",
  url: "http://www.like-every-other-rule.com",
  fields: [{
Normal field definition:
    selector: "input[type=text]",
    value: "the value"
  }, {
Importing the shared rule:
    import: "these field definitions are shared"
  }, {
Another field definition ...
    selector: "#some-id",
    value: "a value"
  }]
}];

In the above example the input[type=text] would be filled with like always since the imported field definition comes last.
