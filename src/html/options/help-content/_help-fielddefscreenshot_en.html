Taking screenshots

You can take screenshots of the page at any time if you want.

There are four ways to trigger screenshots:

  • Inside a field definition
  • At the end of the last field definition
  • Automatically at every step of a workflow
  • Manually via Libs.h.screenshot

To trigger a screenshot from inside a field definition, simply add screenshot: true or screenshot: "some_filename":

fields: [{
  ...
Setting screenshot to true will execute this field definition and then take a screenshot and save it to disk.
  screenshot: true
Setting screenshot to a string will save a screenshot under that name.
The .jpg extension will be added automatically.
  screenshot: "save_with_this_name"
 
}]

Taking a screenshot at the end of the field definitions is primarily important if the rule is part of a workflow.

This can be used to document every step of a running workflow.

var rules = [{
  ...
Setting screenshot to true will take a screenshot at the very end of the rule execution.
  screenshot: true
 
}]

If you want to take screenshots on every workflow step there is an easy way to do so:

In the workflow panel check the "Save a screenshot for every workflow step" option and save the workflow.

Last but not least: Taking a screenshot via Libs.h.screenshot

This helper function can be used inside value functions and takes a screenshot whenever you want.

fields: [{
  selector: ".some-class",
  value: function($element, $data) {
Save a screenshot as "save_with_this_name.jpg"
    Libs.h.screenshot("save_with_this_name");
  }
 
}]
