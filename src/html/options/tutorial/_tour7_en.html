It fills the two field with random data using chance.js.

This is the important part when using workflows.

You need to click/submit the form so that the workflow can continue.

Here it is done by using Libs.h.click on the submit button.

Same for the rule for the second page.

Let's create a workflow using both rules.

Currently there are no workflow defined so lets create one.

Click to the right on "create new workflow".

This is the name of the workflow. It will be displayed in the popup menu in blue like so:
new workflow (1 steps)

Here you will find a list of all defined rules to choose from for the workflow.

They are displayed prefixed with their corresponding tab name.

Choose "Tour 7 : Workflows - Step 1" and press the "add to workflow" button to the right.

Now add the second rule to the workflow.

Choose "Tour 7 : Workflows - Step 2" from the rule list and press the "add to workflow" button again.

Now the workflow is ready.

Since you took care to (Libs.h.)click those submit buttons in every rule the workflow can continue to work even if the URL changes.

You can change the order of rules in the workflow by using the handles on the left of each step to drag-drop them into place.

But for now just save the workflow.

This button comes in handy if a workflow gets stuck when executing.

This can happen for various reasons.

To cancel stuck workflows you can press this button.
