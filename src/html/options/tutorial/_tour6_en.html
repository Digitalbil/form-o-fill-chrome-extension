This rule has two field definitions.

The first is for the input[type=text] field.

Nothing new here.

In this field definition it gets more interesting.

It specifies the input[type=submit] field as the target of the action.

The value for this field definition is a function that simply clicks on the element it is targeted at thereby submitting the form.

This is the common way to submit forms.

Because writing that value function is tedious, Form-O-Fill has a helper function that simplifies the usage.

I changed the rule to one using the helper function.

Instead of writing a function() you can simply use Lib.h.click as the value for the "value".

Note the missing round brackets at the end.

This is the most important feature you will need for workflows to function correctly.
