The first rule defines two fields. This is basic Form-O-Fill.

The rule for the second page uses the import keyword to reference the first rule.

References are always the name of the rule.

If you reference a non-existing rule the extension will complain :)

You can place the import everywhere inside field definitions and they will be imported in place.

If you want to you can also use multiple imports.

Just queue them up as multiple
{ "import": "some rule name" } blocks.

Shared rules can remove duplication. We all like DRY don't we?

That's all. Have fun.
