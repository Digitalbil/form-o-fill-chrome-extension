This is how a value function looks like.

Instead of using a simple value it defines a function.

The value function will receive two parameters.

The first is the DOM element that will be filled by the rule.

In this case it is the <input type="text"> element.

To make working with this element easier, Form-O-Fill will convert this native DOM element into a jQuery DOM element so you can do things like $e.addClass("some-class");

The second parameter (data) contains the data returned from before functions (See Tour 4)

This can be used to fill the field with pre-generated or calculated data.

Whatever you return from the value function will be filled into the form field.

If you return null the field will be left untouched and will not be filled.
