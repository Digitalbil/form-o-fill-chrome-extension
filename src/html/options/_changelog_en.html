Changelog Form-O-Fill

★ Version 2.2.2

  • Workflows graduated from beta
  • Removed export buttons for rules and workflows. Please use the combined button to export both workflows and rules together

★ Version 2.2.1

  • moment.js and chance.js are now loaded on demand. Reduces the memory footprint.
  • Save and load arbitary data using the storage functions in the context object.
    Use it to storage values between workflow steps.

★ Version 2.1.1

  • Fix bug with throbber overlay not showing

★ Version 2.1.0

  • Use the new Libs.halt() function to cancel a workflow/rule from inside before functions.

★ Version 2.0.0

  • Added workflows. You can now automate multiple steps sequentially after one another. This works even after a page load/reload. With this automating complete form workflows is easy.
  • Introduce the Libs.h namespace for value function helper. First helper is Libs.h.click which clicks the matched dom element

★ Version 1.6.1

  • Moved from callback to promise for context.findHtml()
    To convert from callback to promise change
    context.findHtml("selector", function(html) { ... your code ... }); to
    context.findHtml("selector").then(function(html) { ... your code ... });

★ Version 1.6.0

  • Added .findHtml() function to the context object

★ Version 1.5.1

  • Fix a potential chrome bug with flickering or too small popup.
  • Report missing rules when using shared field definitions.

★ Version 1.5.0

  • New feature: Shared field definitions (like rspec shared_examples). Import common field definitions into a rule in any place you want.

★ Version 1.4.1

  • Implemented error reporting for before functions
  • Popup shows rules sorted by name

★ Version 1.4.0
